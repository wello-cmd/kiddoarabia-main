## 2024-05-18 - Adding `aria-label` to icon-only buttons
**Learning:** Found an accessibility issue pattern across this app where `aria-label` was missing for several interactive icon-only elements such as the social links in the footer, open/close options in the AI Bot feature, and navigation icons in the header. The app has bilingual support ('en' and 'ar'), so `aria-label` and `aria-expanded` attributes needed to be conditionally rendered based on the currently selected language using the context or a localized string directly.
**Action:** When adding icon-only buttons (`variant="ghost" size="icon"` or explicitly rounded icon buttons), always provide a descriptive `aria-label` attribute and consider whether `aria-expanded` is required. Furthermore, when the project supports localization like in `Kiddo Arabia`, the `aria-label` string must support localization.
