## 2026-04-11 - Adding ARIA labels for icon-only buttons
**Learning:** Found multiple icon-only buttons (like Social Links, Search, and Mobile Menu Toggle) and standard input fields (Newsletter Input) that were missing accessible `aria-label` attributes, impacting screen reader usability. The `Index Page` test suites failed after mock changes, which required using `getAllByText` / `getAllByRole` over `getByText` / `getByRole` when matching components due to multiple instances existing within the `TestWrapper`. Additionally, wrapping tests consuming `Helmet` elements required an encapsulating `HelmetProvider` for context.
**Action:** Always map available descriptive labels to `aria-label` during map iterations (e.g. `social.label` -> `aria-label`) and provide descriptive `aria-label` text directly on standard inputs.
